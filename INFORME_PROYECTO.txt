SECCIÓN 1: CONCEPTOS GENERALES DEL PROYECTO
================================================================================

1.1. DESCRIPCIÓN GENERAL

El proyecto es un sistema integral web de gestión administrativa y financiera 
diseñado para organizaciones que necesitan controlar facturas, presupuestos, 
gastos y detalles contables. El sistema centraliza la información de múltiples 
procesos como facturación, autorización de gastos, registro contable y control 
de pagos.

1.2. PROPÓSITO PRINCIPAL

Automatizar y centralizar los procesos de:
  • Gestión de facturas de proveedores
  • Distribución y control de presupuestos por centros de costo
  • Seguimiento de gastos y presupuesto disponible
  • Autorización multinivel de documentos financieros
  • Registro contable y análisis fiscal
  • Control y seguimiento de pagos

1.3. USUARIOS OBJETIVO

  • Gerentes y supervisores de centros de costos
  • Personal de Finanzas y Contabilidad
  • Directores de diferentes áreas (Recursos Humanos, Administrativo)
  • Analistas fiscales y contables
  • Administradores del sistema

1.4. ALCANCE GEOGRÁFICO Y OPERATIVO

  • Soporte multiusuario simultáneo
  • Manejo de múltiples centros de costo
  • Información de clientes (empresa o persona natural)
  • Datos contables incluyendo DIAN (Colombia)
  • Moneda: Configurable (por defecto en pesos colombianos)

1.5. MODELO DE DATOS JERÁRQUICO

La aplicación utiliza una estructura de datos en 4 niveles:

  Nivel 1: Centro de Costo (Proyecto/Departamento)
    └── Presupuesto Total Asignado
    
  Nivel 2: Categorías de Presupuesto (Rubros)
    └── Recursos Humanos, Logística, Marketing, etc.
    └── Distribución porcentual del presupuesto
    
  Nivel 3: Items Detallados
    └── Compras, Contrataciones, Servicios
    └── Montos individuales y descripción
    └── Estados: Pendiente → Aprobado → Ejecutado
    
  Nivel 4: Seguimiento y Ejecución
    └── Gastos reales vs presupuestado
    └── Alertas automáticas
    └── Reportes detallados

SECCIÓN 2: DESCRIPCIÓN DE MÓDULOS Y COMPONENTES

2.1. MÓDULO DE AUTENTICACIÓN Y CONTROL DE ACCESO

Concepto: Gestiona el acceso seguro de usuarios al sistema mediante 
credenciales y sesiones.

Componentes Técnicos:
  • modelo: User.js
  • controlador: auth.controller.js
  • middleware: auth.js
  • rutas: auth.routes.js

Características:
  • Registro de usuarios con encriptación bcrypt (10 rondas)
  • Autenticación con JWT (JSON Web Tokens)
  • Gestión de roles: admin, manager, user, analyst
  • Validación de permisos por rol
  • Tokens con expiración configurable
  • Registro de último login de usuarios

Flujo de Seguridad:
  1. Usuario proporciona email y contraseña
  2. Contraseña se valida contra hash almacenado
  3. Se genera JWT firmado con clave secreta
  4. Cliente envía JWT en cada solicitud (Bearer Token)
  5. Middleware valida JWT en rutas protegidas
  6. Se verifica que usuario tenga rol requerido


2.2. MÓDULO DE GESTIÓN DE USUARIOS

Concepto: Administra la creación, modificación y eliminación de usuarios 
del sistema.

Componentes Técnicos:
  • modelo: User.js
  • controlador: user.controller.js
  • rutas: user.routes.js

Características:
  • Crear usuarios con validación de email único
  • Asignar roles y permisos
  • Activar/desactivar usuarios
  • Búsqueda y filtrado de usuarios
  • Cambio de contraseña
  • Auditoría de cambios (created_at, updated_at)

Estructura de Usuario:
  - ID (UUID)
  - Nombre (first_name, last_name)
  - Email (único)
  - Contraseña (hasheada)
  - Rol (admin, manager, user, analyst)
  - Estado activo/inactivo
  - Timestamps de auditoría


2.3. MÓDULO DE GESTIÓN DE FACTURAS

Concepto: Core del sistema. Maneja la captura, validación y ciclo 
completo de facturas de proveedores.

Componentes Técnicos:
  • modelo: Invoice.js
  • controlador: invoice.controller.js
  • rutas: invoice.routes.js
  • tipos: InvoiceType.js

Características Principales:
  • Número de factura único por proveedor
  • Cálculo automático de impuestos (IVA configurablle)
  • Descuentos por línea o total
  • Clasificación de facturas reembolsables vs operacionales
  • Adjuntos de archivos PDF/XML

Ciclo de Vida Completo de la Factura:

  CAPTURA INICIAL:
    - Número de factura
    - Proveedor y tipo de factura
    - Centro de costo asignado
    - Empleado responsable
    - Fecha de emisión y vencimiento
    - Montos: Subtotal, Impuestos, Descuentos, Total
    - Descripción y observaciones

  AUTORIZACIONES (Sistema de Aprobación Multinivel):
    - Aprobación Directa Administrativa
    - Aprobación Directa Upstream (Autoridades)
    - Aprobación Directa RRHH
    - Aprobación Directa Finanzas
    - Aprobación Directa General
    [Cada uno: SI/NO con usuario y fecha]

  REGISTRO CONTABLE:
    - Municipio de registro
    - Tipo de documento (Factura, Nota Crédito, etc.)
    - Número de documento asignado
    - DIAN (Servicio de Impuestos Colombianos)
    - Observaciones contables
    - Usuario registrador

  ANÁLISIS CONTABLE/FISCAL:
    - Validación de sellado correcto
    - Fecha de revisión
    - Archivo XML adjunto
    - Observaciones técnicas
    - Usuario revisor analista

  CONTROL DE PAGO:
    - Fecha de pago realizado
    - Archivo de recibo de pago
    - Monto pagado (puede diferir del total)
    - Observaciones de pago
    - Procesado por

Estados del Documento:
  - Borrador (en captura)
  - Capturada (lista para autorización)
  - En Revisión (esperando aprobaciones)
  - Aprobada (autorizada, lista para contabilidad)
  - Registrada (asentada en contabilidad)
  - Analizada (validada fiscalmente)
  - Pagada (dinero transferido)
  - Cancelada (anulada)

Campos Financieros:
  • Subtotal: Base calcular impuestos
  • Impuesto (IVA u otros): Calculado automáticamente
  • Descuento: Puede ser línea o total
  • Total: Subtotal + Impuesto - Descuento
  • Monto Pagado: Puede ser parcial o total


2.4. MÓDULO DE GESTIÓN DE PRESUPUESTOS

Concepto: Sistema jerárquico para definir, distribuir, ejecutar y 
monitorear presupuestos por centro de costo.

Componentes Técnicos:
  • modelos: 
    - CostCenter.js (Centros de Costo)
    - BudgetCategory.js (Categorías/Rubros)
    - BudgetSubcategory.js (Subcategorías)
    - BudgetItem.js (Items detallados)
  • controlador: budget.controller.js
  • rutas: budget.routes.js

Estructura de Centro de Costo:
  - ID único (UUID)
  - Nombre descriptivo
  - Presupuesto Total asignado
  - Información del Cliente:
    * Nombre del Cliente
    * Tipo Cliente (Persona Natural / Jurídica)
    * Cédula o NIT
    * Número de Contrato

Categorías Predefinidas (Rubros):
  - Recursos Humanos (50%)
  - Logística (20%)
  - Reembolsables (10%)
  - Contratos (10%)
  - Imprevistos (8%)
  - Otros (2%)

Flujo de Presupuesto:

  PASO 1 - ASIGNACIÓN INICIAL:
    Centro de Costo: Empresa ABC
    Presupuesto Total: $5,000,000
    
  PASO 2 - DISTRIBUCIÓN POR CATEGORÍA:
    Recursos Humanos:  $2,500,000 (50%)
    Logística:         $1,000,000 (20%)
    Reembolsables:       $500,000 (10%)
    Contratos:           $500,000 (10%)
    Imprevistos:         $400,000 (8%)
    Otros:               $100,000 (2%)
    
  PASO 3 - ITEMS DETALLADOS POR CATEGORÍA:
    Recursos Humanos:
      - Contratación Gerente:     $600,000 [Pendiente]
      - Contratación Auxiliares:  $300,000 [Aprobado]
      - Capacitación:              $70,000 [Ejecutado]
      - Seguro Médico:             $30,000 [Ejecutado]
      
  PASO 4 - SEGUIMIENTO Y CONTROL:
    Presupuesto Total:    $5,000,000
    Ejecutado:            $2,150,000 (43%)
    Disponible:           $2,850,000 (57%)
    Estado:               En ejecución

Estados de Items Presupuestarios:
  - Pendiente: Aún no se ha gestionado
  - Aprobado: Autorizado para ejecutar
  - Ejecutado: Ya se realizó el gasto
  - Cancelado: Item descartado

SECCIÓN 3: EVALUACIÓN DE SEGURIDAD
================================================================================

3.1. EVALUACIÓN GENERAL

Calificación Seguridad: ⭐⭐⭐⭐ (4 de 5 estrellas)

El sistema implementa medidas de seguridad robustas para una aplicación 
empresarial. Tiene protecciones básicas a intermedias implementadas, con 
algunos puntos que podrían mejorarse para alcanzar máxima seguridad.

3.2. FORTALEZAS DE SEGURIDAD

✅ Autenticación Robusta:
   - Uso de JWT para manejo de sesiones
   - Contraseñas hasheadas con bcrypt (10 rondas)
   - Validación de tokens en cada solicitud protegida
   - Expiración de tokens configurable
   - Tokens firmados criptográficamente

✅ Control de Acceso:
   - Sistema de roles (admin, manager, user, analyst)
   - Middleware de autorización verificando permisos
   - Protección de rutas por nivel de acceso
   - Separación de responsabilidades por rol

✅ Headers de Seguridad:
   - Helmet.js implementado (protege contra XSS, Clickjacking, etc.)
   - CORS configurado (no acepta tokens de cualquier origen)
   - Content Security Policy (CSP)

✅ Encriptación de Datos:
   - Contraseñas encriptadas
   - JWT firmados con clave secreta
   - Conexión a BD con credenciales en variables de entorno

✅ Validación de Entrada:
   - Express Validator en rutas
   - Sanitización de datos
   - Escape en queries SQL
   - UUID para IDs (generas contra IDOR)

✅ Protección SQL:
   - Consultas parametrizadas (prepared statements)
   - Pool de conexiones MySQL seguro
   - No hay SQL injection visible en el código

✅ Cifrado de Datos en Tránsito:
   - HTTPS recomendado en producción
   - JWT Bearer tokens
   - CORS restrictivo

✅ Auditoría:
   - Campos de auditoría: created_by, updated_at
   - Registro de usuarios que realizan acciones
   - Timestamps en cambios de estado


3.3. ÁREAS QUE PODRÍAN MEJORARSE

⚠️ Nivel bajo - Implementable fácilmente:

  • Rate Limiting: Express-rate-limit está instalado pero necesita 
    configuración más refinada (especialmente en login)
    
  • Validación de Archivo: Middleware de upload.js podría tener validaciones 
    más estrictas de tipo MIME y tamaño
    
  • Logging de Seguridad: Winston instalado pero logs de intentos fallidos 
    de acceso podrían ser más detallados
    
  • HTTPS: En producción debe usarse obligatoriamente (usar 
    redirect HTTP → HTTPS)
    
  • Session Timeout: Implementar cierre automático por inactividad
    
  • Dos Factores (2FA): No está implementado (recomendable para usuarios 
    administrativos)

⚠️ Nivel medio - Recomendable agregar:

  • Encriptación de Datos Sensibles: NIT, Cédula, Datos bancarios podrían 
    cifrarse en reposo (AES-256)
    
  • Audit Trail: Registro detallado y inmutable de todas las acciones 
    (especialmente cambios en facturas)
    
  • Web Application Firewall (WAF): En producción usar WAF (AWS WAF, 
    Cloudflare, etc.)
    
  • Backup Automático: Plan de backup y recuperación ante desastres
    
  • Versionado de API: /api/v1/ para compatibilidad hacia atrás


3.4. CONCLUSIÓN SOBRE SEGURIDAD

La aplicación tiene un enfoque de seguridad SÓLIDO apropiado para 
una aplicación empresarial típica. Las fortalezas clave son:

  • Autenticación y autorización bien implementadas
  • Protección contra ataques comunes (SQL injection, XSS)
  • Datos sensibles protegidos
  • CORS y headers de seguridad configurados

Para alcanzar máxima seguridad (S de 5 estrellas), se recomienda 
implementar: 2FA, cifrado de datos en reposo, WAF, audit trail 
más detallado y rate limiting más agresivo.

ES SEGURO para usar en producción con ambiente controlado (acceso 
restringido, backups, HTTPS obligatorio).
